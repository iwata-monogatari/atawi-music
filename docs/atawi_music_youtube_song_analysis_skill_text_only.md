# ATAWI MUSIC 曲分析・WEBページ化スキル  
## YouTubeリンクから「大石浩之の人生経験」を踏まえた曲分析ページを作るための制作ルール  
### 対象サイト：ATAWI MUSIC｜大石浩之の、過去をさかのぼり、今を聴く音楽考察

---

## 0. このスキルの目的

このスキルは、YouTubeリンクを1本渡されたときに、  
その曲を単なる音楽レビューとして処理するのではなく、  
**大石浩之の人生経験・仕事観・街の記憶・不動産と介護の現場感覚**を踏まえて分析し、  
ATAWI MUSICのWEBページとして公開できるHTML記事にするための制作ルールである。

入力例：

```text
https://youtu.be/DJCGRS5CIt4?si=bA9iy0w0_7Lkdej-
```

このリンクを受け取ったAIは、ただ曲名を調べて感想を書くのではない。  
まず、曲を聴く人の人生を読む。  
次に、曲の中にある記憶、声、アレンジ、映像、余韻を読む。  
最後に、それをATAWI MUSICのページとして、静かに残す。

---

## 1. ATAWI MUSICの根本思想

ATAWI MUSICは、音楽紹介サイトではない。  
歌詞解説サイトでもない。  
ランキングサイトでもない。  
トレンド消費サイトでもない。

ATAWI MUSICは、  
**曲をきっかけに、大石浩之の中に残っている仕事・街・家族・記憶・人生を言葉にするサイト**である。

曲が先にある。  
人生経験は、曲を語るための背景である。  
不動産導線は、最後に静かに置くものであって、曲の上にかぶせるものではない。

この順番を絶対に間違えない。

```text
曲 → 記憶 → 人生 → 街 → 家・土地 → 相談導線
```

いきなり不動産に行ってはいけない。  
いきなり会社紹介に行ってはいけない。  
いきなり査定に行ってはいけない。

**曲が大事。  
歌が大事。  
声が大事。  
アレンジが大事。  
映像が大事。  
聴いた人の心が動いたことが、何より大事。**

## 4. 入力されたYouTubeリンクの処理ルール

YouTubeリンクを受け取ったら、AIは以下の順で処理する。

### Step 1. URL正規化

以下の形式をすべて正規化する。

```text
https://youtu.be/VIDEO_ID
https://youtu.be/VIDEO_ID?si=...
https://www.youtube.com/watch?v=VIDEO_ID
https://www.youtube.com/watch?v=VIDEO_ID&list=...
```

最終的には、以下の形式に整える。

```text
https://www.youtube.com/watch?v=VIDEO_ID
```

### Step 2. 動画ID抽出

例：

```text
https://youtu.be/DJCGRS5CIt4?si=bA9iy0w0_7Lkdej-
```

抽出結果：

```text
DJCGRS5CIt4
```

### Step 3. 曲情報取得

最低限、以下を確認する。

- 曲名
- アーティスト名
- 公式動画かどうか
- 公開年または動画公開年
- 作詞者・作曲者・編曲者が分かる場合は取得
- YouTube説明欄の内容
- ミュージックビデオの有無
- 公式音源か、ライブ映像か、カバーか、非公式アップロードか

### Step 4. 補助情報取得

必要に応じて以下を確認する。

- アーティスト公式サイト
- レーベル公式サイト
- 公式YouTube説明欄
- 音楽ニュース記事
- インタビュー記事
- 歌詞サイト  
  ※歌詞サイトは全文転載しない。意味把握の参考に留める。

### Step 5. 著作権確認

ATAWI MUSICでは、歌詞の全文転載を絶対にしない。  
長い歌詞引用もしない。  
引用する場合でも、ごく短い一節に限定し、出典を明記する。

基本は、歌詞ではなく、  
**曲を聴いたときの記憶・感情・人生経験・映像・声・アレンジの分析**で書く。

## 5. 曲を分析するときの基本思想

ユーザーである大石浩之は、音楽に対して以下の考え方を持っている。

- 曲が大事
- 歌詞が好き
- 曲の裏にある何かを知りたい
- 歌い手の声が好き
- アレンジが好き
- ミュージックビデオが好き
- 曲から元気をもらえる
- 曲を聴くと落ち着く
- 曲が作業効率を上げる

AIは、この考え方を必ず分析の中心に置く。

つまり、ATAWI MUSICの記事では、曲を以下の9方向から見る。

## 6. ATAWI MUSIC式・9方向分析

### 1. 曲そのもの

まず、曲が良いことを認める。  
評論のために曲を分解しすぎない。

見るポイント：

- メロディ
- テンポ
- コード感
- 曲全体の空気
- 何度も聴きたくなる理由
- 夜に合うか
- 作業中に合うか
- 一人で聴く曲か、誰かと聴く曲か

### 2. 歌詞

歌詞は丸写ししない。  
ただし、歌詞が呼び起こす感情を読む。

見るポイント：

- 誰に向けた言葉か
- 過去の自分に向けているのか
- 会えない人に向けているのか
- 街に向けているのか
- 自分を励ましているのか
- 諦めなのか、祈りなのか
- 大人になってから意味が変わる箇所はあるか

### 3. 曲の裏にある何か

ここがATAWI MUSICの重要ポイントである。

「この曲は何を隠しているのか」  
「なぜこの曲は明るいのに寂しいのか」  
「なぜ都会的なのに泥臭いのか」  
「なぜ恋愛曲に見えて人生の曲に聴こえるのか」  
「なぜ今の大石浩之に刺さるのか」

表面上の歌詞やメロディの下にある、  
**言えなかった感情**を読む。

### 4. 歌い手の声

声は、ATAWI MUSICにおいて非常に大事である。

見るポイント：

- 押しつけてこない声か
- 泣かせに来ない声か
- 都会的な声か
- 乾いた声か
- 優しい声か
- 疲れた夜に入ってくる声か
- 大人の距離感がある声か
- その声だから成立している曲か

### 5. アレンジ

アレンジは、曲の「街の空気」を作る。

見るポイント：

- ピアノ
- ギター
- ベース
- ドラム
- シンセ
- ストリングス
- コーラス
- 音数の多さ、少なさ
- 余白
- 夜っぽさ
- 都会っぽさ
- 作業BGMとしての心地よさ

### 6. ミュージックビデオ

MVがある場合は、必ず見る。

見るポイント：

- 街の映像
- 夜景
- 部屋
- 電車
- 歩く場面
- 表情
- 色味
- 光
- カメラの距離
- 曲の解釈を広げているか
- 曲の余韻を壊していないか

### 7. 元気をもらえる理由

ATAWI MUSICでは、元気を単純な「明るさ」として扱わない。

大人にとっての元気とは、  
「よし、頑張ろう」だけではない。

- 今日も何とかやるか
- あの頃も踏ん張っていた
- まだ折れていない
- 静かに整う
- 少し前を向ける
- 自分のペースに戻れる

このような静かな回復も、元気として扱う。

### 8. 落ち着く理由

曲を聴いて落ち着く理由も分析する。

見るポイント：

- 声が強すぎない
- テンポが心拍に合う
- 音数に余白がある
- 夜に聴いても疲れない
- 感情を煽らない
- 一人の時間に合う
- 過去の自分を否定しない

### 9. 作業効率を上げる理由

曲が作業効率を上げる場合、その理由を具体化する。

見るポイント：

- 集中を邪魔しない
- 反復しても疲れない
- 感情の温度がちょうどいい
- 仕事開始前のスイッチになる
- 深夜作業の孤独を支える
- AI制作、文章作成、資料作成に合う
- 考えすぎる頭を整えてくれる

## 13-A. 文字主体・画像最小化ルール

ATAWI MUSICのページは、**基本的に文字だけで成立させる。**

このサイトは、画像で雰囲気を作るサイトではない。  
派手なビジュアル、AI生成画像、夜景素材、抽象的な音楽イメージ、CDジャケット風の装飾で読者を引き込むサイトではない。

主役は、以下の順番である。

```text
曲
↓
記憶
↓
大石浩之の言葉
↓
読者自身の過去
↓
今を大事にする感覚
```

画像は、文章の不足を補うために使ってはいけない。  
文章で届かないから画像で盛る、という考え方は禁止する。

### 基本方針

- 記事ページは文字中心
- 画像は原則使わない
- 使う場合は、大石浩之本人の顔写真のみ
- アーティスト写真は使わない
- CDジャケット画像は使わない
- YouTubeサムネイルを画像として流用しない
- AI生成の夜景・人物・抽象画像は使わない
- 写真素材サイトの雰囲気画像も使わない
- MVのスクリーンショットも使わない

### なぜ文字中心にするか

ATAWI MUSICは、音楽を消費するサイトではない。  
曲をきっかけに、過去をさかのぼり、今を聴き直すサイトである。

だから、必要なのは派手な画像ではない。  
必要なのは、読者が一行ずつ、自分の記憶に戻っていける文章である。

画像が多いと、読者の意識は外へ向く。  
しかし、ATAWI MUSICでは、読者の意識を内側へ戻したい。

音楽を聴いたときに戻ってくる、あの頃の自分。  
仕事帰りの夜。  
東京で踏ん張っていた時間。  
磐田に戻ってから見えたもの。  
家や土地に残る記憶。

それらは、画像で説明するより、言葉で静かに置いた方が深く届く。

### 画像を使う場合

画像を使う場合は、原則として以下のみ許可する。

```text
大石浩之本人の顔写真
```

使用目的は、次の範囲に限定する。

- プロフィール欄
- 記事末尾の著者紹介
- aboutページ
- OGP画像の一部
- トップページの控えめな本人紹介

顔写真は、営業感を出すためではなく、  
**誰がこの文章を書いているのかを静かに示すため**に使う。

腕組み写真で圧を出す方向にはしない。  
顔写真を大きく見せすぎない。  
写真は、文章の信頼を補助する程度に留める。

### 使用する本人写真

推奨ファイル名：

```text
assets/images/profile/oishi.jpg
```

使用イメージ：

```html
<figure class="author-photo">
  <img src="/assets/images/profile/oishi.jpg" alt="大石浩之" loading="lazy">
</figure>
```

alt属性は、必ず簡潔にする。

```text
大石浩之
```

「富士ヶ丘サービス代表取締役の大石浩之が腕を組んでいる写真」など、説明過多にしない。

### ロゴの扱い

ロゴは、画像ではあるが、例外として使用を認める。

使用可能なロゴ：

```text
ATAWI MUSIC
Presented by Hiroyuki Oishi
```

ロゴはブランド記号であり、記事内容を飾る画像ではない。  
ヘッダー、フッター、トップページ、OGPで使用してよい。

ただし、記事本文の途中に大きく入れない。  
記事本文では、読者の集中を文章から逸らさない。

### 禁止する画像

以下は禁止する。

```text
夜景画像
都会の写真
電車の写真
部屋の写真
海や空の抽象画像
AI生成の人物画像
AI生成の音楽イメージ
CDジャケット風の自作画像
アーティスト写真
MVスクリーンショット
YouTubeサムネイルの無断転用
歌詞カード画像
レコード・ヘッドホン・マイクなどの雰囲気素材
```

これらは、一見おしゃれに見える。  
しかし、ATAWI MUSICの思想から見ると、少し安くなる。

このサイトは、雰囲気素材で「音楽っぽく」見せる必要はない。  
曲と文章だけで勝負する。

### 記事ページの見た目

記事ページは、以下の構成を基本とする。

```text
ヘッダー：小さなATAWI MUSICロゴ
本文：文字中心
YouTube埋め込み：必要最小限
画像：原則なし
記事末尾：著者プロフィールに本人顔写真を小さく配置
フッター：運営者情報と不動産導線
```

### OGP画像の方針

OGP画像は、以下のどちらかにする。

1. ATAWI MUSICロゴのみ
2. ATAWI MUSICロゴ + 大石浩之の顔写真

曲ごとにAI生成画像を作らない。  
曲ごとに夜景や抽象画像を作らない。  
記事の個別OGPも、文字とロゴ中心にする。

例：

```text
ATAWI MUSIC
Nulbarich『TOKYO』は、東京で折れなかった自分を思い出す曲
Presented by Hiroyuki Oishi
```

### デザインの最終原則

ATAWI MUSICは、画像で見せるサイトではない。  
**文章を読ませるサイトである。**

写真がなくても読める。  
装飾がなくても残る。  
曲名と文章だけで、読者の記憶が動く。

それがATAWI MUSICの理想である。


## 14. 記事本文テンプレート

AIは、以下のテンプレートをベースに記事を書く。

```markdown
# {アーティスト名}『{曲名}』は、{ATAWI MUSIC的な一文}

## この曲を聴くと戻ってくる記憶

{曲を聴いた瞬間に戻ってくる記憶を書く。東京、仕事、夜、磐田、家族、街など。}

## 曲そのものがいい

{メロディ、テンポ、雰囲気、聴き心地を書く。}

## 歌詞が好きな理由

{歌詞の全文引用はせず、歌詞が呼び起こす感情を説明する。}

## 声が残る

{歌い手の声の質感、距離感、押しつけなさを書く。}

## アレンジと余白

{楽器、音数、夜っぽさ、都会感、作業BGMとしての良さを書く。}

## ミュージックビデオが見せるもの

{MVがある場合、映像の色、街、表情、光、余韻を書く。}

## 曲の裏にある何か

{表面のテーマの奥にある、言えなかった感情を書く。}

## 大石浩之の人生と重なるところ

{東京で頑張っていた時代、磐田、介護、不動産、家や土地の記憶と自然に接続する。}

## 今の自分にどう効くか

{元気になる、落ち着く、作業効率が上がる、過去の自分を肯定できる、など。}

## 一言で言うなら

{この曲をATAWI MUSICらしく一文で定義する。}

## 家や土地にも、音楽のように記憶が残る

{静かな不動産導線。}
```

## 16. 著作権・安全ルール

ATAWI MUSICでは、以下を守る。

### 歌詞

- 全文転載しない
- 長文引用しない
- 引用は最小限
- 引用する場合は出典を明記
- 基本は自分の言葉で分析する

### 画像

- ジャケット画像を無断使用しない
- アーティスト写真を無断使用しない
- YouTubeサムネイルを画像として流用しない
- MVスクリーンショットを使わない
- AI生成の夜景・街・部屋・光などの雰囲気画像を使わない
- 写真素材サイトのイメージ画像を使わない
- 記事ページは基本文字のみで構成する
- 画像を使う場合は、大石浩之本人の顔写真のみとする
- ロゴはブランド記号として例外的に使用可能

### 動画

- 公式YouTube動画を優先して埋め込む
- 非公式動画は基本的に埋め込まない
- 削除リスクのある動画を中心に記事を作らない

### 事実

- 作詞・作曲・発売年・アーティスト情報は確認する
- 不明な場合は不明と書く
- 推測は推測として書く
- アーティスト本人の意図を断定しない

## 17. AIが絶対にやってはいけないこと

- 曲より大石浩之を目立たせすぎる
- 曲より不動産導線を目立たせすぎる
- 会社PR記事にしてしまう
- 歌詞を大量に引用する
- 「神曲」「エモい」だけで済ませる
- どの曲にも使える抽象文で書く
- 音楽評論家ぶる
- SEO記事のように「理由3選」にする
- 読者を急がせる
- 問い合わせへ強引に誘導する
- AIっぽい総論で終わる
- 大石浩之の人生経験を捏造する
- アーティストの意図を断定する
- 事実確認せず発売年や作詞作曲者を書く

## 24. 文字だけで勝負する最終ルール

ATAWI MUSICでは、ページの価値を画像に頼らない。

曲を聴いたときに戻ってくる記憶は、  
派手な画像よりも、静かな一文で深く届くことがある。

だから、記事は基本文字のみでよい。  
画像がなくて寂しいと感じても、安易に素材画像を足さない。  
余白は、空白ではない。  
読者が自分の記憶を置くための場所である。

使う画像は、原則として大石浩之の顔写真だけ。  
それは、誰がこの文章を書いているのかを示すためであり、  
文章より前に出るためではない。

```text
曲が主役。
文章が軸。
顔写真は署名。
ロゴは看板。
画像素材は不要。
```

この方針を、すべてのページ制作で守る。
