## 2. 記事執筆基準 (atawi-music-article-skill)

### 2-1. 最重要ルール
- **本文文字数**: 必ず **3,000文字以上** とします。
- **H2見出し数**: 本文内のH2見出し（`<h2>`）は **最大4個まで** とします（「参考リンク」「書いた人」などの共通見出しはカウントに含めません）。見出しが多すぎると落ち着いて読めなくなるため、似た論点は1つの見出しにまとめ、4個以内で構成します。

### 2-2. 文章のトーン＆マナー
- やさしく、静かで、余韻がある大人の文章。
- 単なる感想文や、AI特有の一般論（「心に響く」「おすすめ」等の凡庸な単語の多用）で終わらせず、その曲が持つ固有の引力を丁寧に言語化します。
- **歌詞の扱い**: 歌詞の全文転載や長い引用は厳禁です。歌詞に触れる場合は、必要最小限の短い引用にとどめ、描かれている感情や時間を自身の言葉で考察します。

### 2-3. 大石浩之の人生文脈との接続
- 音楽をきっかけに、大石が東京で頑張っていた頃の記憶、磐田の風景、介護や不動産の仕事の中で見てきた家族や土地の歴史などを自然に織り交ぜます。
- ただし、自分語りが強すぎて曲が脇役にならないよう、常に曲を引き立てるための素材としてエピソードを用います。

## 3. 大石セレクション星評価 (atawi-music-oishi-selection-article-skill)

### 3-1. 3大評価軸と星評価基準
記事内、および `data/songs.json` 内に、以下の3項目の星評価（★1〜★5）を付与します。
1. **曲がいい**: メロディ、コード進行、構成、アレンジ、ボーカルの乗り方、イントロやアウトロの完成度。
2. **歌詞がいい**: 言葉選び、テーマ性、物語性、年齢を経ることで変わる解釈、余白の美しさ。
3. **MVがいい**: 公式MV（または公式ライブ映像）の構図、色、演出、編集、曲との親和性。
   - インスト曲（歌詞がない楽曲）については、`lyrics` の評価軸自体を省略し、2項目のみで評価します。

### 3-2. 主視点「大石セレクション視点」の選定
- 3項目のうち、最も評価（星の数）が高いものを「大石セレクション視点」として記事冒頭と songs.json の `selection` に定義します。
- 星の数が同点の場合は、「その曲を初めて人に紹介するときに最も惹かれる要素」を主視点に選定します。
- 定量的な星の数にかかわらず、定性的な理由により低い評価の軸をあえて主視点に選ぶこともあります（例：曲=5, MV=4 だが、映像の役割が非常に重要であるためMVを主視点とする）。

### 3-3. 記事内の評価ブロック記述例
```html
<div class="oishi-rating">
  <p class="oishi-rating-headline">大石セレクション：曲がいい ★★★★☆</p>
  <ul class="oishi-rating-list">
    <li>曲がいい：★★★★☆</li>
    <li>歌詞がいい：★★★☆☆</li>
    <li>MVがいい：★★★☆☆</li>
  </ul>
  <p class="oishi-rating-reason"><strong>選定理由：</strong>（具体的な選定理由を記述する）</p>
</div>
```

## 4. YouTube記事化・公開ワークフロー (atawi-music-youtube-article)

### 4-1. 登録制限（YouTube公式チェック）
以下のいずれかのチャンネルが公開している公式MV・公式音源のみを対象とします。
- アーティストの公式チャンネル (Official Channel)
- 所属レーベルの公式チャンネル
- 公式の音楽プロジェクトやトピック (Topic) チャンネル
※ ファンによる二次アップロード、無許可のリリックビデオ、権利関係が不明な動画は掲載不可とします。

### 4-2. 事前調査ステップ (Mandatory)
記事執筆前に必ずWeb検索等で以下の要素を調査します。
1. **制作背景**: 作詞・作曲・編曲の布陣、制作エピソード、タイアップ情報、リリース背景。
2. **チャート成績**: オリコン順位、売上、ストリーミング再生数などの客観的データ。
3. **音楽的特徴**: コード進行、メロディ展開、転調、ボーカルの質感、楽器アレンジ。
4. **惹きつける理由**: なぜこの曲が長く愛され、聴き手の感情を動かすのかの考察。
※ 調査された確証事実は「◯◯のインタビューによれば」と記載し、推測データは「〜とされる」とぼかします。

### 4-3. 記事末尾の不動産 funnel CTA ブロック
各記事の末尾、**「参考リンク」のすぐ下、かつ「書いた人(author-box)」の上**に、以下の不動産案内ブロックを必ず1回だけ挿入します。
```html
<div class="cta">
  <p>（音楽と記憶を結ぶ、静かで控えめな導入文。例：デビュー曲が後から評価を積み上げていくように、家や土地にも、時間を経てようやく見えてくる価値があります。）</p>
  <p>静岡県磐田市周辺で、相続した実家・空き家・土地建物の整理にお悩みの方は、富士ヶ丘サービスまでご相談ください。</p>
</div>
```
※ 過去の `aside.fs-cv`（インラインスタイル付きの枠線カード）を使用したスタイルから、サイト全体のCSSクラス `.cta` に最適化された上記記述に完全移行します。

## 5. サイト登録・技術的運用手順 (atawi-music-skill)

### 5-1. 重複URLガード
新規記事登録時、必ずYouTube動画ID（URLから抽出した一意のID。`watch?v=` と `youtu.be/` の両方を正規化）を抽出し、既存の `data/songs.json` 内の全データと重複チェックを行います。すでに同一IDが登録されている場合は、登録を拒否し既存ページを案内します。

### 5-2. songs.json への登録
以下のJSON構造に従って `data/songs.json` に追記します。
```json
{
  "song_id": "furuuchi-toko-001",
  "artist_id": "furuuchi-toko",
  "song_title": "somewhere in TOKYO",
  "release_year": 1996,
  "era": "1990年代",
  "genre": "J-POP",
  "lyricist": "古内東子",
  "composer": "古内東子",
  "arranger": "小松秀行",
  "youtube_url": "https://www.youtube.com/watch?v=XXXXXX",
  "article_url": "articles/furuuchi-toko-001.html",
  "themes": ["tokyo", "night", "city"],
  "moods": ["night-groove"],
  "oishi_context": "東京で頑張っていた頃の記憶...",
  "status": "published",
  "recommended": true,
  "rating": {
    "song": 5,
    "lyrics": 4,
    "mv": 3,
    "selection": "song"
  }
}
```
※ インスト曲は `rating.lyrics` を省略します。

### 5-3. 検索インデックスの再構築 (Critical)
`data/songs.json` を更新した後は、必ずリポジトリのルートで以下のスクリプトを実行して `data/search-index.json` を再生成します。これを行わないと、トップページや一覧に新規曲が反映されません。
```bash
node tools/build-search-index.mjs
```

### 5-5. ビルド＆デプロイ手順
1. リポジトリ (`C:\Users\fujig\Documents\GitHub\atawi-music`) で変更ファイルをコミット・プッシュします。
2. 変更ファイルを本番公開用クリーンフォルダ (`C:\Users\fujig\tmp\atawi-music-public`) へ個別に同期（コピー）します。
